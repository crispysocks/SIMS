## 二、应用入口：main.py

### 2.1 文件作用

整个后端程序的**启动文件**。运行 `uvicorn app.main:app` 时，Python 会加载这个文件，创建 FastAPI 应用实例。

### 2.2 核心工作

| 步骤 | 说明 |
|------|------|
| 1. 创建 FastAPI 实例 | `FastAPI(title='SIMS 学生管理系统', version='0.1.0')` |
| 2. 配置 CORS | 允许前端 `http://localhost:5173` 跨域访问（从 `.env` 读取） |
| 3. 注册路由 | 把 `api/` 下所有模块的路由挂到主应用上 |
| 4. 根路径接口 | 访问 `/` 返回 `{'message': 'SIMS backend is running'}` |
| 5. 启动事件 | 调用 `init_db()` 自动创建表和默认管理员 |

### 2.3 路由注册清单

```python
app.include_router(agent_router)           # /agent/...
app.include_router(auth_router)            # /auth/...
app.include_router(students_router)        # /students/...
app.include_router(classes_router)         # /classes/...
app.include_router(scores_router)          # /scores/...
app.include_router(employment_router)      # /employment/...
app.include_router(employment_v2_router, prefix='/v2')  # /v2/employment/...
app.include_router(statistics_router)      # /statistics/...
app.include_router(teachers_router)        # /teachers/...
app.include_router(users_router)           # /users/...
```

> **注意**：`employment_v2_router` 加了 `prefix='/v2'`，所以访问路径是 `/v2/employment/...`。

## 三、认证与权限：dependencies.py

### 3.1 文件作用

处理**"谁可以访问哪个接口"**的问题。FastAPI 的依赖注入系统会自动调用这里的函数，把当前用户信息传给接口函数。

### 3.2 核心类：CurrentUser

```python
class CurrentUser(BaseModel):
    username: str           # 当前登录用户的用户名
    roles: list[str]        # 角色列表，如 ['admin']、['teacher']
```

### 3.3 核心函数：get_current_user

**解析当前登录用户**，使用 JWT Token 认证：

| 方式 | 说明 |
|------|------|
| JWT Token 认证 | 读取 `Authorization: Bearer <token>`（标准方式） |

**解析流程**：
1. 检查 `Authorization: Bearer <token>` 头部
2. Token 不存在 → 返回 401
3. Token 无效或过期 → 返回 401
4. Token 中无用户信息 → 返回 401

### 3.4 核心函数：require_role

**创建角色校验器**。用法示例：

```python
# 只有管理员能访问
@router.get('/admin-only', dependencies=[Depends(require_role(['admin']))])

# 管理员和老师都能访问
@router.get('/teacher-and-admin', dependencies=[Depends(require_role(['admin', 'teacher']))])
```

**原理**：检查 `current_user.roles` 和 `allowed_roles` 是否有交集，没有就返回 403。

1. 环境变量（如 `DB_HOST=localhost`）

| `DB_USER` | `root` | 数据库用户名 |
| `DB_PASSWORD` | `123456` | 数据库密码 |
| `DB_POOL_SIZE` | `5` | 连接池大小 |
| `DB_POOL_RECYCLE` | `3600` | 连接回收时间（秒） |

## 五、数据库管理：core/database.py

### 5.1 文件作用

数据库的**连接管理、会话提供、表结构初始化、示例数据加载**。

### 5.2 核心对象

| 对象 | 类型 | 说明 |
|------|------|------|
| `engine` | `Engine` | 数据库连接引擎，管理所有连接 |
| `SessionLocal` | `sessionmaker` | 会话工厂，用来创建数据库会话 |
| `Base` | `declarative_base()` | ORM 基类，所有模型类都要继承它 |

### 5.3 自动创建数据库

```python
def _ensure_database_exists():
    """如果数据库不存在，自动创建（使用 utf8mb4 编码）"""
```

**为什么需要？** 新环境部署时，数据库可能还没创建，这个函数会在程序启动时自动建好。

### 5.4 数据库会话提供：get_db()

```python
def get_db():
    db = SessionLocal()
    try:
        yield db      # 交给接口函数使用
    finally:
        db.close()    # 确保无论成功与否都会关闭连接
```

**用法**：在接口函数的参数里写 `db: Session = Depends(get_db)`，FastAPI 会自动注入。

### 5.5 初始化数据库：init_db()

**启动时自动执行**，做四件事：

1. **导入所有模型模块** → 让 SQLAlchemy 知道有哪些表要创建
2. **创建所有表** → `Base.metadata.create_all(bind=engine)`
3. **创建默认管理员账号** → 用户名 `admin`，密码 `123456`，角色 `admin,teacher`
4. **加载示例数据** → 如果 `LOAD_SAMPLE_DATA=True`

### 5.6 示例数据加载

```python
def _load_sample_data():
    """从 app/data/sample_data.json 加载测试数据"""
```

**包含的数据类型**：教师、班级、学生、成绩、就业信息。

**防重复机制**：导入前检查数据库是否已有相同数据，有则跳过。

**自动关闭开关**：导入完成后，自动把 `.env` 里的 `LOAD_SAMPLE_DATA=True` 改成 `False`。

## 八、Agent 接口：api/agent.py

### 8.1 文件作用

给**智能助手（Agent）**提供的专用接口，支持：
1. **灵活 SQL 查询** —— 可以查任意数据（但只能查，不能改）
2. **数据保存** —— 把查询结果或传入的数据保存到 Excel 或数据库

### 8.2 接口清单

| 接口 | 方法 | 路径 | 权限 |
|------|------|------|------|
| SQL 查询 | POST | `/agent/sql/query` | admin/teacher |
| 保存数据 | POST | `/agent/save` | admin/teacher |

### 8.3 SQL 查询接口

**请求格式**：
```json
{
  "sql": "SELECT * FROM students WHERE isdeleted = 0 LIMIT 5",
  "params": {}
}
```

**安全限制**：
- 必须以 `SELECT` 开头
- 禁止包含：`insert`, `update`, `delete`, `drop`, `truncate`, `alter`, `create`, `grant`, `revoke`
- 使用**单词边界匹配**（`\b`），避免字段名如 `isdeleted` 被误判

**响应格式**：
```json
{
  "message": "查询成功",
  "data": {
    "columns": ["student_no", "name"],
    "rows": [{"student_no": "2021001", "name": "张三"}],
    "row_count": 1
  }
}
```

### 8.4 保存数据接口

支持 **4 种组合**：

| source_type | target_type | 说明 |
|-------------|-------------|------|
| `sql` | `xlsx` | SQL 查询结果 → Excel 文件 |
| `sql` | `db` | SQL 查询结果 → 数据库表 |
| `data` | `xlsx` | JSON 数据 → Excel 文件 |
| `data` | `db` | JSON 数据 → 数据库表 |

**示例 A：SQL 导出 Excel**
```json
{
  "source_type": "sql",
  "sql": "SELECT * FROM students WHERE isdeleted = 0",
  "target_type": "xlsx",
  "file_path": "./exports/students.xlsx"
}
```

**示例 B：JSON 写入数据库**
```json
{
  "source_type": "data",
  "data": [{"student_no": "2021001", "name": "张三"}],
  "target_type": "db",
  "table_name": "temp_import"
}
```

> 写入数据库时使用 `if_exists='append'` 模式，表不存在会自动创建。

## 九、认证接口：api/auth.py

### 9.1 文件作用

处理用户的**注册和登录**。

### 9.2 接口清单

| 接口 | 方法 | 路径 | 说明 |
|------|------|------|------|
| 注册 | POST | `/auth/register` | 创建新账号，默认角色 `teacher` |
| 登录 | POST | `/auth/login` | 验证身份，返回 JWT token |

### 9.3 注册流程

1. 检查用户名是否已存在 → 存在则返回 400
2. 创建新用户，密码用 MD5 加密
3. 保存到数据库
4. 生成 JWT token
5. 返回 token + 用户名 + 角色列表

### 9.4 登录流程

1. 根据用户名查找用户 → 不存在则返回 401
2. 验证密码（MD5 比对）→ 错误则返回 401
3. 生成 JWT token
4. 返回 token + 用户名 + 角色列表

## 十一、统计分析接口：api/statistics.py + services/statistics.py

### 11.1 文件作用

提供**各种统计查询**，用于数据分析和报表展示。只查询不修改，所以不需要权限控制。

### 11.2 接口清单

| 接口 | 路径 | 说明 |
|------|------|------|
| 按年龄筛选 | `/statistics/age-filter?age=25` | 查询年龄 >= 指定值的学生 |
| 班级性别统计 | `/statistics/class-gender` | 每个班的男女生人数 |
| 学霸筛选器 | `/statistics/always-above?score=80` | 每次考试都高于某分数的学生 |
| 挂科统计 | `/statistics/failed-twice` | 两次及以上不及格的学生 |
| 班级平均分 | `/statistics/class-avg-score` | 各班级每次考试的平均分 |
| 高薪榜 | `/statistics/top-salary` | 薪资排名前 5 的学生 |
| 个人就业时长 | `/statistics/student-offer-duration` | 每个学生从入学到拿 offer 的天数 |
| 班级平均就业时长 | `/statistics/class-offer-duration` | 每个班的平均就业天数 |

### 11.3 关键技术点

**班级性别统计** —— 使用 SQL `CASE` 语句在查询中做条件计数：
```python
func.sum(case((Student.gender == '男', 1), else_=0)).label('male_count')
```

**学霸筛选器** —— 使用子查询找出"有低分记录的学生"，再用 `not_in` 排除：
```python
failed_subquery = db.query(Score.student_no).filter(Score.score <= score)
Student.student_no.not_in(failed_subquery)
```

**就业时长** —— 使用 MySQL `TIMESTAMPDIFF` 计算天数差：
```python
func.timestampdiff(func.text('day'), Employment.employment_open_time, Employment.offer_time)
```

## 十七、关键设计决策与注意事项

### 17.1 认证方式

系统使用 **JWT Token 认证**（`Authorization: Bearer <token>`），标准方式，适用于所有环境。

### 17.2 软删除

所有业务表都有 `isdeleted` 字段（0=正常，1=已删除）。
- 查询时**必须**加上 `WHERE isdeleted = 0`
- 删除操作实际上是更新 `isdeleted = 1`

### 17.3 密码安全

- 数据库中**绝不存储明文密码**
- 使用 MD5 哈希（当前），建议生产环境升级到 bcrypt
- 默认管理员密码是 `123456`，部署后务必修改

### 17.4 数据库连接池

- 连接池大小：5（`DB_POOL_SIZE`）
- 连接回收：3600秒（`DB_POOL_RECYCLE`）
- 使用 `pool_pre_ping=True` 检测连接是否存活

### 17.5 文件上传

- 上传目录：`./uploads`（相对路径）
- 最大文件大小：5MB
- 注意：目录不会自动创建，需要手动创建
