## 沃林学⽣管理系统需求⽂档 

## **1.** 项⽬概述 

本项⽬旨在开发⼀个基于FastAPI的学⽣管理系统，提供学⽣基本信息管理、考核成绩管理和就业管理三⼤核⼼功能模 块。系统将采⽤RESTful API设计，⽀持前后端分离架构。 

## **2.** 功能需求 

## **2.1** 学⽣基本信息管理模块 

## 学⽣信息增改查 

- 创建新学⽣记录（学⽣编号、学⽣班级、学⽣姓名、籍贯、毕业院校、专业、⼊学时间、毕业时间、学历、 顾问编号、年龄、性别） 

- 查询学⽣信息（⽀持按编号、姓名、班级等条件筛选） 

- 更新学⽣信息 

逻辑删除学⽣ 

## **2.2** 学⽣考核成绩管理模块 

## 成绩录⼊与管理 

- 按考核序次录⼊考核成绩（学⽣编号、考核序次、成绩） 

- 成绩修改与删除 

## **2.3** 学⽣就业管理模块 

## 就业信息管理 

- 记录学⽣就业状态（学⽣编号、学⽣姓名，学⽣班级，就业开放时间、offer下发时间、就业公司名称、就 业薪资） 

- 按照学⽣编号，公司名字，⼯资范围查询学⽣就业信息 

- 修改和删除学⽣就业信息 

- 学⽣姓名(冗余)、学⽣班级(冗余) 属于冗余字段，可⾃⾏设计表结构 

## **2.4** 班级管理模块 

班级信息管理（班级编号， 开课时间， 班主任， 授课⽼师等信息） 

## **2.5** ⽼师管理模块 

⽼师信息管理（⽼师信息， 带班信息等）

## **2.6** 统计分析模块 

## **2.6.1** 基本信息统计 

- 查询所有超过30岁的学员的信息 

- 统计每个班级的⼈数以及男⽣⼥⽣的⼈数 

## **2.6.2** 成绩统计 

- 查询每次考试成绩都在80分以上的学⽣的编号，姓名和成绩 

- 查询有两次以上不及格的学⽣的姓名，班级和不及格成绩 

- 统计每次考试每个班级的平均分，按照从⾼到低排序 

## **2.6.3** 就业统计 

- 统计就业薪资最⾼的前五名学⽣的姓名，班级和就业时间，就业公司 

- 统计每个学⽣的就业时⻓（offer下发时间-就业开放时间） 

- 统计每个班级的平均就业时⻓（只统计进⼊就业阶段的学⽣，也就是有就业开放时间）

## **4. API** 设计概要 

## 系统接⼝要遵从 **Restful** 规范，并且考虑前端便利性和⽤⼾体验，以及效率 

## **4.1** 学⽣基本信息端点 

```
GET    /students          # 获取学生列表
POST   /students          # 创建新学生
GET    /students/{id}     # 获取特定学生信息
PUT    /students/{id}     # 更新学生信息
```

## **4.2** 成绩管理端点 

```
GET    /score/{stu_id}              # 获取学生成绩
POST   /score/                      # 录入成绩
PUT    /score/update                # 修改指定学生的某次成绩
POST   /score/delete                # 删除指定学生的某次成绩
```
