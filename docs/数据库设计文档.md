# 学生管理系统数据库设计文档

## 1. 数据库设计概述

### 1.1 设计原则
- 符合第三范式(3NF)，消除数据冗余
- 主键采用业务编号（自然键），不使用代理键
- 不设计冗余字段，需要关联查询时通过JOIN获取
- 使用逻辑删除标记isdeleted
- 添加审计字段created_at, updated_at

## 5. 典型查询SQL示例

### 5.1 基本信息统计
```sql
-- 查询所有超过30岁的学员
SELECT * FROM students 
WHERE age > 30 AND isdeleted = 0;

-- 统��每个班级的人数以及男女人数
SELECT c.class_name,
       COUNT(s.student_no) AS total_count,
       SUM(CASE WHEN s.gender = '男' THEN 1 ELSE 0 END) AS male_count,
       SUM(CASE WHEN s.gender = '女' THEN 1 ELSE 0 END) AS female_count
FROM classes c
LEFT JOIN students s ON c.class_no = s.class_no AND s.isdeleted = 0
WHERE c.isdeleted = 0
GROUP BY c.class_no;
```

### 5.2 成绩统计
```sql
-- 查询每次考试成绩都在80分以上的学生
SELECT s.student_no, s.name
FROM students s
JOIN scores sc ON s.student_no = sc.student_no AND sc.isdeleted = 0
WHERE s.isdeleted = 0
GROUP BY s.student_no
HAVING MIN(sc.score) >= 80;

-- 查询有两次以上不及格的学生
SELECT s.student_no, s.name, COUNT(*) AS fail_count
FROM students s
JOIN scores sc ON s.student_no = sc.student_no AND sc.isdeleted = 0
WHERE sc.score < 60 AND s.isdeleted = 0
GROUP BY s.student_no
HAVING COUNT(*) >= 2;

-- 统计每次考试每个班级的平均分
SELECT sc.exam_no, c.class_name, AVG(sc.score) AS avg_score
FROM scores sc
JOIN students s ON sc.student_no = s.student_no
JOIN classes c ON s.class_no = c.class_no
WHERE sc.isdeleted = 0 AND s.isdeleted = 0 AND c.isdeleted = 0
GROUP BY sc.exam_no, c.class_no
ORDER BY sc.exam_no;
```

### 5.3 就业统计
```sql
-- 统计就业薪资最高的前五名学生
SELECT e.*, s.student_no, s.name
FROM employment e
JOIN students s ON e.student_no = s.student_no
WHERE e.isdeleted = 0 AND s.isdeleted = 0
ORDER BY e.salary DESC
LIMIT 5;

-- 统计每个学生的就业时长(天)
SELECT s.student_no, s.name,
       DATEDIFF(e.offer_time, e.employment_open_time) AS employment_days
FROM employment e
JOIN students s ON e.student_no = s.student_no
WHERE e.isdeleted = 0 AND s.isdeleted = 0 
  AND e.employment_open_time IS NOT NULL 
  AND e.offer_time IS NOT NULL;

-- 统计每个班级的平均就业时长
SELECT c.class_name,
       AVG(DATEDIFF(e.offer_time, e.employment_open_time)) AS avg_employment_days
FROM employment e
JOIN students s ON e.student_no = s.student_no
JOIN classes c ON s.class_no = c.class_no
WHERE e.isdeleted = 0 AND s.isdeleted = 0 AND c.isdeleted = 0
  AND e.employment_open_time IS NOT NULL 
  AND e.offer_time IS NOT NULL
GROUP BY c.class_no;
```

## 6. 建表SQL脚本

```sql
-- 创建数据库
CREATE DATABASE IF NOT EXISTS student_management CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE student_management;

-- 老师表
CREATE TABLE teachers (
    teacher_no VARCHAR(20) PRIMARY KEY,
    name VARCHAR(50) NOT NULL,
    gender ENUM('男','女') NOT NULL,
    phone VARCHAR(20),
    email VARCHAR(100),
    id_card VARCHAR(18),
    birthday DATE,
    hire_date DATE,
    position VARCHAR(50),
    subject VARCHAR(50),
    isdeleted TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 班级表
CREATE TABLE classes (
    class_no VARCHAR(20) PRIMARY KEY,
    class_name VARCHAR(50) NOT NULL,
    class_open_time DATE NOT NULL,
    head_teacher_no VARCHAR(20),
    instructor_no VARCHAR(20),
    description VARCHAR(500),
    isdeleted TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 添加班级表外键
ALTER TABLE classes 
ADD CONSTRAINT fk_classes_head_teacher 
FOREIGN KEY (head_teacher_no) REFERENCES teachers(teacher_no) ON DELETE SET NULL;

ALTER TABLE classes 
ADD CONSTRAINT fk_classes_instructor 
FOREIGN KEY (instructor_no) REFERENCES teachers(teacher_no) ON DELETE SET NULL;

-- 学生表
CREATE TABLE students (
    student_no VARCHAR(20) PRIMARY KEY,
    class_no VARCHAR(20) NOT NULL,
    name VARCHAR(50) NOT NULL,
    birth_place VARCHAR(100),
    graduate_school VARCHAR(100),
    major VARCHAR(50),
    entrance_time DATE NOT NULL,
    graduate_time DATE,
    education VARCHAR(20),
    advisor_name VARCHAR(50),
    age INT,
    gender ENUM('男','女') NOT NULL,
    phone VARCHAR(20),
    id_card VARCHAR(18),
    isdeleted TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 添加学生表外键
ALTER TABLE students 
ADD CONSTRAINT fk_students_class 
FOREIGN KEY (class_no) REFERENCES classes(class_no) ON DELETE RESTRICT;

-- 考核成绩表
CREATE TABLE scores (
    student_no VARCHAR(20) NOT NULL,
    exam_no INT NOT NULL,
    exam_name VARCHAR(50),
    score DECIMAL(5,2) NOT NULL,
    exam_date DATE,
    remark VARCHAR(200),
    isdeleted TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (student_no, exam_no)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 添加成绩表外键
ALTER TABLE scores 
ADD CONSTRAINT fk_scores_student 
FOREIGN KEY (student_no) REFERENCES students(student_no) ON DELETE CASCADE;

-- 就业信息表
CREATE TABLE employment (
    student_no VARCHAR(20) PRIMARY KEY,
    employment_status ENUM('待业','在聘','已离职') DEFAULT '待业',
    employment_open_time DATETIME,
    offer_time DATETIME,
    company_name VARCHAR(100),
    salary DECIMAL(10,2),
    position VARCHAR(50),
    work_location VARCHAR(100),
    resign_date DATE,
    remark VARCHAR(200),
    isdeleted TINYINT DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

-- 添加就业表外键
ALTER TABLE employment 
ADD CONSTRAINT fk_employment_student 
FOREIGN KEY (student_no) REFERENCES students(student_no) ON DELETE CASCADE;
```

## 7. 设计说明

### 7.1 方案B设计说明
- 所有表使用业务编号（student_no, teacher_no, class_no）作为主键
- 不使用自增ID代理键，减少表关联时的字段冗余
- 外键关联直接使用业务编号，简化JOIN操作

### 7.2 冗余字段处理
- 不设计冗余字段，所有关联数据通过JOIN查询获取

### 7.3 逻辑删除
- 所有业务表添加isdeleted字段(0=正常,1=已删除)
- 所有查询默认加上 `isdeleted = 0` 条件

### 7.4 审计字段
- created_at: 记录创建时间
- updated_at: 记录最后更新时间(自动更新)

### 7.5 约束说明
- 成绩取值范围建议应用层控制(0-100)
- 就业状态字段使用枚举类型
- 外键采用级联删除策略保护数据完整性
